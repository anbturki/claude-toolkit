# Git Commit

Create a commit following project conventions.

## Steps

1. Check status and diff:
   ```bash
   git status
   git diff --staged
   git diff
   ```

2. Stage specific files (prefer over `git add .`):
   ```bash
   git add <specific-files>
   ```

3. Review recent commits for style:
   ```bash
   git log --oneline -5
   ```

4. Create commit with HEREDOC:
   ```bash
   git commit -m "$(cat <<'EOF'
   <type>: <description>

   <optional body>
   EOF
   )"
   ```

## Commit Types

| Type | Use Case |
|------|----------|
| `feat` | New feature |
| `fix` | Bug fix |
| `refactor` | Code restructuring |
| `chore` | Maintenance tasks |
| `docs` | Documentation |
| `style` | Formatting changes |
| `test` | Test changes |
| `perf` | Performance improvements |

## Examples

```bash
# Feature
git commit -m "$(cat <<'EOF'
feat: add contact import functionality

Support CSV file upload for bulk contact creation
EOF
)"

# Bug fix
git commit -m "$(cat <<'EOF'
fix: resolve duplicate phone number validation

Check for duplicates before creating contact
EOF
)"

# Refactor
git commit -m "$(cat <<'EOF'
refactor: extract auth logic into shared module

Move authentication business logic to shared package
EOF
)"
```

## Rules

1. **No AI references** - Never mention Claude, AI, or assistants
2. **No Co-Authored-By** - Do not include AI co-author lines
3. **Focus on what and why** - Not how
4. **Imperative mood** - "add feature" not "added feature"
5. **Keep title under 72 chars**
6. **Stage specific files** - Avoid `git add .` or `git add -A`
7. **Never skip hooks** - No `--no-verify`

## Do NOT Include

- `Co-Authored-By: Claude` or any AI reference
- Emoji in commit messages
- Time estimates
- "Generated by" references
